## Problems

### 1. Free LLM API instability

During the implementation of Chat Service Hardening,
I encountered multiple problems with free LLM models.

Problems:
- Some models returned 404 because the free endpoint was unavailable.
- Some models returned 429 because the provider tate limit was reached.
- The same model could work at one moment and fail later.

Solution:
- Added model fallback mechanism.
- Separated model configuration from business logic.
- Used environment variables to control model selection.

Learning:
External AI services are unstable dependencies.
AI engineers need to design systems that can handle failures.

## Backend Architecture Understanding

Today I improved my understanding of backend architecture.

Route layer:
- Responsible for HTTP request handling.
- Should not contain business logic.

Service layer:
- Responsible for business logic.
- Handles LLM calls and AI workflow.

Core layer:
- Responsible for application-level configuration and AI behavior.
- Contains settings and prompts.

This separation makes the system easier to maintain and extend.

## Bug Analysis

Problem:
Fallback model configuration was not loaded.

Root cause:
Environment variable name typo.

Expected:
OPENAI_FALLBACK_MODEL

Actual:
OPENAL_FALLBACK_MODEL

Debug process:
1. Added debug logs.
2. Checked runtime configuration.
3. Compared .env variables with Python settings.
4. Found configuration mismatch.

Lesson:
Configuration problems are common in backend development.
Never assume configuration is loaded correctly.
Verify runtime values.
