## Sprint Task 13 - LLM Parameters Configurable

## Reflection

Today I made the LLM parameters configurable through the `.env` file instead of hardcoding them in the source code.

During development, I encountered tow practical issues.

First, I found that the application was still using the default values because the environment variable names did not match the configuration keys exactly. This taught me the importance of verifying configuration values with debug logs.

Second, I learned that modifying the `.env` file does not take effect immediately. Since the settings are loaded only when the application starts, I had to restart the FastAPI server to reload the environment variables.

This task helped me better understand configuration management in backend applications.
