## Reflection

Today I introduced a Prompt Builder into the project.

Although the generated messages are the same as before, I learned that prompt construction should be separated from the LLM calling logic.

This design makes the code easier to maintain and allows futures such as user profiles, conversation summaries, and RAG knowledge retrieval to be added without changing the chat service.

I also experienced a short moment of confusion at the beginning, bug after understanding the architecture, I was able to complete the implementation independently. This gave me more confidence in reading requirements and designing code based on responsibilities.
