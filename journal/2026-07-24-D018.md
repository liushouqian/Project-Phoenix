# Task 16 Journal

## Goal

Make conversation window configurable instead of hardcoding the value.

## Done

- Added CONVERSATION_WINDOW configuration in .env.
- Updated config.py to load this value.
- Updated chat_service.py to use settings.CONVERSATION_WINDOW.
- Verified different conversation window sizes.

## Problem

During testing, I found that the LLM response was not always consistent with the actual history sent to the model.

For example, when I set the conversation window to 10, the model answered that it could see message1 to message12.

## Solution

I checked the debug logs and found that the actual history sent to the model was limited correctly.

The issue was caused by the LLM generating an answer based on its understanding, not because the system sent wrong data.

## What I Learned 

The LLM response is not always the source of truth.

During debugging, application logs and actual data flow are more reliable than the model's generated answer.

I also learned that conversation history management is an important part of AI application design.
