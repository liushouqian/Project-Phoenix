## Problems Encountered

The main challenge today was understanding why the model only remembered part of the conversation.

At first, I thought it was a bug. After testing multiple conversation rounds, I realized the system was intentioinally limiting the context window to improve performance and reduce token usage. This helped me understand the difference butween persistent memory (SQLite) and the context window sent to the LLM.

## What I Learned

Today I learned an important AI engineering concept.

The application stores the complete conversation history in SQLite, but it should not send the entire history to the LLM. Instead, only the most recent messages are included in each request. This reduces latency, lowers token costs, and matches how production AI applications usually work.

I also learned how SQL queries ( ORDER BY , LIMIT ) can be used to control the amount of conversation history passed to the model.

## Reflection

Today's task helped me understand that AI engineering is not only about calling an API. Designing how conversation history is stored and how much context is provided to the model is also an important part of building production AI systems.
