## Sprint Task 11 - Conversation Manager

## Reflection

Today's task introduced a Conversation Manager layer.

At first, it looked like an unnecessary wrapper because it only forwarded function calls to the conversation store.

However, I learned that this layer improves the architecture by separating business logic from storage logic.

In the future, features such as caching, conversation summarization, vector databases, or memory management can be added inside the manager without changing the service layer.

This task helped me understand the importance of clean architecture and maintainability.
